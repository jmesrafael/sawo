🔄 UPDATE LOCAL BUTTON (Rotate Icon)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Hover to see:
  ┌──────────────────────────────────────────────────────────────────┐
  │ Update Local Cache — Syncs only changed products. Fast and       │
  │ efficient. Use this after editing or creating products.          │
  └──────────────────────────────────────────────────────────────────┘

  What it does:
    ✅ Compares Supabase with local cache
    ✅ Only syncs changed/new/deleted products
    ✅ Updates local JSON files
    ✅ Works in seconds

  When to use:
    • After editing a product
    • After creating a new product
    • After deleting a product
    • After changing categories/tags
    • Most common operation


⬇️  FULL REFRESH BUTTON (Download Icon)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Hover to see:
  ┌──────────────────────────────────────────────────────────────────┐
  │ Full Refresh Cache — Deletes all cached data and re-downloads    │
  │ everything from Supabase. Use this if cache seems outdated or    │
  │ corrupted.                                                        │
  └──────────────────────────────────────────────────────────────────┘

  What it does:
    ✅ Wipes entire local cache
    ✅ Re-downloads ALL data from Supabase
    ✅ Rebuilds all JSON files from scratch
    ✅ Fixes stale/corrupted cache

  When to use:
    • Cache seems outdated
    • Data looks wrong
    • After major bulk operations
    • Before important deployments
    • Troubleshooting issues


🧹 STORAGE CLEANUP BUTTON
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Hover to see:
  ┌──────────────────────────────────────────────────────────────────┐
  │ Storage Cleanup — Remove orphaned files. Scans both image and    │
  │ PDF storage buckets to find and delete files that aren't attached│
  │ to any product. Safe to run anytime.                             │
  └──────────────────────────────────────────────────────────────────┘

  What it does:
    ✅ Scans product-images bucket
    ✅ Scans product-pdf bucket
    ✅ Finds files NOT used by any product
    ✅ Deletes orphaned files
    ✅ Frees storage space

  When to use:
    • After deleting products
    • After removing images
    • Monthly maintenance
    • Storage is full
    • Before checking storage costs

  What gets deleted:
    ✅ Files from deleted products
    ✅ Uploaded but never used images
    ✅ Old images replaced in products

  What's safe:
    ✅ All products stay intact
    ✅ Only removes unused files
    ✅ Can preview with "Dry Run"
